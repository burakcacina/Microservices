The "D" in SOLID stands for the Dependency Inversion Principle (DIP).
High-level modules should not depend on low-level modules. Both should depend on abstractions.

The CAP theorem (Consistency, Availability, Partition tolerance) states that a distributed data store can only guarantee two out of three properties simultaneously:


Consistency: All nodes see the same data at the same time.
Availability: Every request receives a (non-error) response, without guarantee that it contains the most recent write.
Partition tolerance: The system continues to operate despite arbitrary message loss or failure of part of the system.
